# VatBrain — Brain-Inspired Memory Architecture for AI Agents

## 设计原理与指导思想

---

### 摘要

当前 AI Agent 的记忆机制普遍采用"线性上下文堆叠"——将历史信息不加区分地塞入 Context Window。这种模式与人脑的记忆机制存在根本性差异：人脑通过多级衰减、关联检索和自我整合，实现对信息的动态管理。本文档提出一套借鉴人脑记忆机制的 AI Agent 记忆架构，核心包括：多级存储架构、"图+向量"复合索引、情境感知检索、错误记忆独立建模、睡眠整合机制，以及基于"遗忘即默认"的权重衰减策略。

---

## 1. 核心理念：遗忘不是 bug，是 feature

人脑记忆系统的核心悖论：**遗忘是默认状态，记住才是例外**。

当前大多数 Agent 记忆方案的设计哲学是"尽量多记、尽量少丢"，这导致索引膨胀、检索噪声增加、关键信号被稀释。人脑的方案恰恰相反——它通过主动遗忘来维持信噪比。

### 设计推论

- **写入门槛极高**：不是所有交互都值得进入长时记忆。只有通过显著性门控的信息才能持久化。
- **衰减是默认路径**：每条记忆从创建起就进入衰减曲线。除非被后续行为"续命"，它将在到达冷却阈值后自然消失。
- **删除是被动的，记住是主动的**：系统不需要决策"该删什么"——自然衰减会处理低价值信息。

这一点是整个架构最底层的设计哲学。

## 2. 多级存储架构

人脑记忆并非一块单一硬盘，而是按保留时间和抽象程度分为多级存储器。

| 人脑机制 | AI Agent 对应 | 技术载体 | 核心职责 |
|---------|-------------|---------|---------|
| 感官记忆 | 原始输入 Buffer | FIFO 队列 | 初筛去噪，剔除语气词等无意义信息 |
| 工作记忆 | 动态 Context Window | LLM 实时压缩的 Task State | 遵循"7±2 法则"，只保留当前任务核心逻辑链 |
| 情境记忆 | 对话历史与事件记录 | 向量数据库（带时间戳） | "什么时候发生了什么" |
| 语义记忆 | 知识、规则与事实 | 图数据库 | "知识是什么"，概念节点与关系 |

### 关键设计

**写入即关联（Link on Write）**：新信息进入长时记忆时，同步执行：(1) 向量定位——寻找语义最接近的存量记忆；(2) 图连接——在新旧信息之间建立带权重的边。这避免了"写入时只管存、检索时再想怎么找"的懒惰模式。

**空间压缩（Abstraction）**：高度重合的信息不应创建新节点，而应更新旧节点并提升其权重。这避免了索引随数据量线性膨胀。

## 3. "图 + 向量"复合存储模型

### 3.1 神经元类比

| 人脑结构 | 工程对应 | 承载信息 |
|---------|---------|---------|
| 神经元 | 图节点 | 实体（函数、API、模块、规则） |
| 突触权重 | 边属性 | 关系强度（调用频率、依赖紧密度） |
| 神经元激活 | 向量相似度 | 语义触发（"长得像"的候选集） |
| 髓鞘化（快路径） | LRU 缓存 | 高频访问的热记忆 |

### 3.2 分工

- **图数据库（拓扑连接）**：关系路径、调用链、依赖关系——精确的结构化查询
- **向量数据库（语义触发）**：模糊匹配、似曾相识——高召回率的语义发现

两者不可互相替代。图负责"知道和谁有关系"，向量负责"找到可能有关的东西"。

### 3.3 复合写入流程

1. 向量定位：寻找语义上最接近的存量记忆候选集
2. 可分离性判别（Pattern Separation）：确认新记忆与候选集的相似是"真相关"还是"巧合相似"
3. 建立连接：在新旧信息之间建立边，strength 根据时序/逻辑相关性动态调整
4. 空间压缩决策：若高度重合 → 更新旧节点；若实质不同 → 创建新节点

第 2 步（可分离性判别）是当前大多数系统缺失的。没有它，两个不同项目的 Redis 超时问题会被错误合并，导致检索时的信息混淆。

## 4. 检索机制：从"找相似的"到"找适用的"

### 4.1 两阶段检索

单阶段语义检索的根本缺陷：向量相似度最高的结果可能来自完全不同的上下文。一个 Python 脚本可能和 Go 后端在向量空间中很接近，但完全不适用。

**第一阶段：情境过滤（Contextual Gating）**

用廉价计算筛除不相关的上下文，将候选集从全库缩小到几百条。

```
情境维度：
├── 硬约束（不匹配直接排除）
│   ├── project_id
│   ├── language / framework
│   └── task_type (debug | feature | refactor | review)
├── 软权重（影响排序但不排除）
│   ├── time_proximity（越近权重越高）
│   ├── active_files（当前文件相关度）
│   └── recent_corrections（最近被纠正过的领域加权）
```

硬约束只设 project_id 和 language 两项，其余为软权重。跨技术栈的通用经验（如"并发问题通常出在锁粒度"）仍有迁移价值。

**第二阶段：语义排序**

在候选集内做向量相似度匹配。此时候选集已经过情境筛选，向量排序才能真正发挥作用。

### 4.2 检索中的反作用：记忆再巩固

人脑的记忆再巩固机制揭示了一个关键事实：**每次回忆都不是只读操作**。记忆被提取时会重新进入不稳定状态，可以被修改。

工程推论：检索不应无副作用。当一条记忆被检索出来给 Agent 使用，而用户随后纠正了 Agent 的行为——这个纠正信号应反向传播到被检索的记忆上，更新它。不是"发现冲突就覆盖"，而是"被用到且被纠正时才更新"。

## 5. 记忆活跃度分层与逻辑屏蔽

大脑并不物理删除过时记忆，而是通过前额叶皮层的抑制机制进行逻辑屏蔽。

### 5.1 活跃度分层

| 状态 | 定义 | 决策权重 |
|-----|------|---------|
| 热记忆 (Active) | 当前任务、最近 2 天修改、明确强调的规范 | 最高，直接参与决策 |
| 温记忆 (Warm) | 近期访问过、同项目不同模块 | 参与排序，不主动注入 |
| 冷记忆 (Dormant) | 已结项项目、三个月前的调试逻辑 | 静默存储，除非显式触发 |

### 5.2 版本覆盖

同一逻辑实体的多版本不采用链表串联，而采用**版本集合 + 时间戳**模型：
- 同一 `entity_group` 下的所有版本共享一个语义标识
- 新版本写入时，旧版本标记 `obsoleted_at` 时间戳
- 检索时通过 `entity_group` 索引直接定位最新版本

这避免了 Superseded_By 链表在频繁更新场景下退化为 O(n) 遍历。

## 6. 权重与衰减机制

### 6.1 为什么简单的 Weight = Frequency / Time_Distance 不够

线性倒数公式至少在四个维度上有缺陷：
- 衰减函数形状：人脑遗忘近似幂律，线性倒数严重偏离
- 频率的时间分布：同一天被调用 10 次 vs 连续 10 天每天 1 次，信息含义不同
- 时间参照点：从创建时间算还是最后访问时间算？两者应共同参与
- 绝对冷却阈值：低于某权重的记忆应从热索引中物理移除

### 6.2 改进方案

**Recency-Weighted Frequency（时间加权频率）**：

```
Effective_Frequency = Σ e^(-λ · (now - access_i))
```

这区分了"一天内密集调用"和"长期稳定调用"——前者频率高但衰减快，后者频率低但累计权重稳。

**双参照时间衰减**：

```
Weight = Effective_Frequency · e^(-α · (now - created_at)) · e^(-β · (now - last_accessed_at))
```

其中 `α` 控制"经验丰富度"衰减（越久远的经验越不可靠），`β` 控制"活跃度"衰减（越久没用的越冷）。

**冷却阈值**：

当 Weight 低于 CoolingThreshold 时，记忆从热索引中移除进入冷存储。冷存储不参与常规检索，仅在显式触发时可用。

### 6.3 重要性不由评分决定，由行为归因推断

不应让 LLM 在当下对每次交互打分——当下的 LLM 无法判断什么对未来重要。重要性的真正信号来自后续行为：

- 被后续检索命中 → +权重
- 引出了用户纠正 → +权重（且标记为"纠错源"，获得更高保护级别）
- 引出了 Agent 行为改变 → +权重
- 从未被回访 → 自然衰减

这与广告系统中的"转化归因"逻辑一致：一次曝光的价值由它最终驱动的转化行为决定，而非曝光时的估算。

## 7. 错误记忆：独立建模

### 7.1 为什么错误记忆需要独立于语义记忆

语义记忆回答"怎么做"（how-to），错误记忆回答"怎么炸的"（how-it-failed）。两者的信息结构完全不同：

| 维度 | 语义记忆 | 错误记忆 |
|-----|---------|---------|
| 触发条件 | "我要用 X" | "我要用 X 且 Y 刚出问题了" |
| 检索时机 | 主动规划时 | 异常/调试/纠正时 |
| 信息密度 | 低（正常路径，熵小） | 高（异常路径，每个 bug 都是独特的） |
| 价值衰减 | 慢（知识相对稳定） | 慢于事件但快于知识 |

### 7.2 数据结构

一个 Pitfall 节点至少包含：

```
(source_entity)          -- 出问题的代码实体
(signature)              -- 错误特征（特征向量/模式描述，非完整 traceback）
(root_cause_category)    -- 根因分类（并发/资源耗尽/配置错误/契约违反/...）
(fix_strategy)           -- 修复策略摘要
(was_user_corrected)     -- 纠错来源（是用户直接纠正 = 高可信）
(occurred_at)            -- 时间戳
```

### 7.3 演进价值：从提醒到预判

错误记忆积累到一定量后，产生第二层价值——**风险预测**。当 Agent 即将修改某个模块时，系统计算该模块及其依赖的"风险得分"（= 历史 Pitfall 密度 × 最近修改时间衰减因子），在 working memory 中预先注入警告。记忆驱动预判。

## 8. 睡眠整合机制

这是当前 AI Agent 最欠缺的一层。大脑在睡眠中进行记忆固化——将白天的事件提炼为知识。

### 8.1 整合过程的三阶段

1. **回顾**：扫描最近（24h）的情境记忆增量，识别高密度修改区域
2. **提炼**：从零散的情境记忆中提取模式，生成候选语义规则
3. **回测**：用候选规则去"预测"近期情境记忆中相关行为。预测准确率低于阈值则不生效，留在待验证状态

第三步是关键的安全阀——整合结果在被信任前必须经过验证。

### 8.2 反事实推理

睡眠阶段不仅回放"发生了什么"，还可以模拟"如果走了另一条路会怎样"。例如：用户连续三次手动修正了 Agent 在同一类问题上的行为 → 生成假设性规则更新，不等第六次纠正。

### 8.3 溯源不可丢弃

整合后的语义记忆必须保留指向源情境记忆的溯源链路。当语义规则被检索但用户表现出纠正行为时，纠正信号通过溯源链反向传播到源记忆。**可逆性是安全的整合和危险的妄断之间的分水岭。**

## 9. 信息的多层抽象：维度坍缩

从原始对话到人格画像，信息经过多层压缩。这个过程的数学本质是：**信息从高维向低维坍缩，保留不变性，丢弃细节**。

### 9.1 三层抽象

| 层级 | 内容 | 存储 |
|-----|------|------|
| Level 1（原始像素） | 原始代码/完整对话 | 对象存储（MinIO/磁盘） |
| Level 2（特征提取） | 结构化元数据（输入输出、状态转换） | 图数据库精简节点 |
| Level 3（语义沉淀） | 逻辑图谱节点（"该模块负责支付路由"） | 图数据库核心节点 |

### 9.2 升层条件：信息增益，而非数量阈值

不应预设"N 条 Level 2 记录触发升层"。而用**信息增益**判断：当多条 Level 2 记录在描述同一模式，且新增记录不显著改变该模式的统计分布 → 模式已稳定，可提炼为 Level 3。若新模式仍在接收显著变体 → 留在 Level 2。

### 9.3 压缩时保留残差

记忆整合时不仅保存摘要文本，同时保存摘要与原始记忆之间的 **embedding 残差向量**。未来检索时若摘要与 query 的相似度不够精确，可通过残差向量判断是否需要"解压"回更原始的记忆层。

## 10. 编码特异性：情境依赖再提取

人脑的一个关键特性：记忆提取强烈依赖于编码时的状态/上下文。你在学东西的环境里回忆它，效果最好。

对 Agent 来说这意味着：存储一条记忆时，不仅存内容，还存当时的**情境向量**（当前任务类型、活跃的仓库/文件、使用的工具链）。检索不是 `query x memory` 的一步操作，而是两步：(1) 匹配情境，缩小候选集；(2) 在候选集内做语义匹配。

这解释了为什么你一打开某个项目，就自然想起该项目的约束——情境帮你做了预过滤。Agent 的设计应体现这个原则。

---

## 11. 源监控与可信度

大脑经常记得信息本身但忘了来源。这在 Agent 记忆中是致命的。

同一条"Redis MaxOpenConns=100"的规则：
- 来源是用户明确指令（高可信）
- 来源是某次成功的调试推断（中等可信）
- 来源是 LLM 推测（低可信）

应有截然不同的**检索优先级**和**可覆盖难度**。源标签不是元数据的装饰——它直接决定冲突时这条记忆是被修改还是被保护。

### 来源分级

| 来源 | 可信度 | 可覆盖条件 |
|-----|--------|-----------|
| 用户显式声明 | 高 | 仅用户显式推翻 |
| 代码 AST 提取 | 高（可验证） | 代码变更自动更新 |
| 调试验证 | 中 | 用户纠正或更新证据 |
| LLM 推断 | 低 | 任何冲突源可覆盖 |

---

## 12. 预测误差：被忽视的学习信号

多巴胺神经元编码的核心是**奖励预测误差**——预期和实际之间的差距。大脑在预测被打破时学习最快。

推论：最高权重的记忆不应是被反复使用的记忆，而应是**让预期落空的记忆**。一个 API 调用的 100 次成功不如 1 次意外失败信息量大。

应引入独立的"惊讶度"维度（Surprise Score），与常规权重并行但独立计算。高惊讶度的事件在衰减曲线上获得更长的半衰期。

## 13. 架构总览

```
                    输入流
                      │
              ┌───────▼───────┐
              │  Sensory Buffer │  ← FIFO，去噪
              └───────┬───────┘
                      │ 显著性门控
              ┌───────▼───────┐
              │ Working Memory │  ← LLM 动态压缩 Task State
              └───────┬───────┘
                      │ 意图边界触发
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
┌───────────┐  ┌───────────┐  ┌───────────┐
│  Vector DB │  │ Graph DB   │  │  Pitfall   │
│ (情境记忆)  │  │ (语义记忆)  │  │ (错误记忆)  │
└─────┬─────┘  └─────┬─────┘  └─────┬─────┘
      │              │              │
      └──────────────┼──────────────┘
                     │
            ┌────────▼────────┐
            │  情境过滤 + 语义  │
            │  两阶段检索       │
            └────────┬────────┘
                     │
            ┌────────▼────────┐
            │  睡眠整合引擎    │  ← 异步，Agent 闲置时运行
            │  回顾→提炼→回测  │
            └─────────────────┘
```

## 14. 设计原则总结

1. **遗忘是默认，记住是例外**：写入长时记忆的门槛极高，衰减是自然路径
2. **检索从"找相似的"到"找适用的"**：先过情境过滤器，再过语义匹配
3. **图管拓扑，向量管语义**：不可互相替代
4. **重要性从行为推断，不从当下评分**：事后归因 > 现场判断
5. **错误记忆独立建模**：坏经验比好经验的熵更高
6. **睡眠整合必须可逆**：溯源链路是安全阀
7. **源标签决定可信度**：同一信息，不同来源 = 不同对待
8. **预测误差是最高价值信号**：让预期落空的事件应获得更长半衰期
9. **压缩保留残差**：丢了细节不可怕，丢了找回细节的路径才可怕
10. **先模式分离，再模式补全**：合并前先确认不可分离
